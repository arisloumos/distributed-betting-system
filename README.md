**Τι έχω υλοποιήσει (The Backend Core):**
*   **SRG (Secured Random Generator):** Ένας αυτόνομος TCP Server που λειτουργεί με το μοντέλο **Producer-Consumer** (χρήση `wait/notify`). Παράγει τυχαίους αριθμούς και τους στέλνει μαζί με ένα **SHA-256 Hash** για ασφάλεια.
*   **Master Node:** Ο ενορχηστρωτής. Δέχεται συνδέσεις από Manager/Player. Χρησιμοποιεί **Hashing** (`hashCode % workers`) για να μοιράζει τα παιχνίδια στους Workers και υλοποιεί τη λογική **MapReduce** για την αναζήτηση παιχνιδιών.
*   **Worker Node:** Η "μονάδα επεξεργασίας". Αποθηκεύει τα παιχνίδια στη μνήμη, επικοινωνεί με τον SRG για τα πονταρίσματα και υπολογίζει κέρδη/ζημιές με βάση τους πίνακες ρίσκου (Low/Medium/High).
*   **Common:** Κοινές κλάσεις (`Game`, `HashUtils`) που χρησιμοποιούνται από όλους για να "μιλάνε" την ίδια γλώσσα.

**Τι πρέπει να γίνει για να κλείσει το ΜΕΡΟΣ Α (Deadline 03/04):**
1.  **JSON Parsing (Manager):** Στο `ManagerApp`, αντί για hardcoded παιχνίδια, πρέπει να προστεθεί κώδικας που διαβάζει ένα αρχείο `.json` (με βιβλιοθήκη όπως η **Gson** ή **Jackson**) και το στέλνει στον Master.
2.  **Filtering (Search):** Στον `WorkerNode`, η αναζήτηση πρέπει να δέχεται φίλτρα (αστέρια, όρια πονταρίσματος, ρίσκο) και να επιστρέφει μόνο τα παιχνίδια που ταιριάζουν.
3.  **Manager Queries (Stats):** Πρέπει να φτιαχτεί μια λειτουργία όπου ο Manager θα ζητάει "Συνολικά κέρδη ανά Πάροχο" ή "ανά Παίκτη". Ο Master θα πρέπει να κάνει **MapReduce** (να ρωτάει όλους τους Workers και να αθροίζει τα αποτελέσματα).

**Τι μένει για το ΜΕΡΟΣ Β (Deadline 10/05):**
1.  **Android App:** Η `DummyPlayer` θα αντικατασταθεί από μια κανονική εφαρμογή Android.
2.  **UI/UX:** Προβολή λογοτύπων, κουμπιά για ποντάρισμα, ασύγχρονη αναζήτηση (Threads στο Android).
3.  **Bonus (Replication):** Αν θέλετε το +20%, πρέπει να υλοποιήσετε το Active Replication (αν πέσει ένας Worker, να υπάρχει αντίγραφο).
